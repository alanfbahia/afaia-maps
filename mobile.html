Como o Modo Offline Funciona

Cada dado tem seu ciclo de vida local antes de ir para a nuvem

Online — sincronizado
Mapas Offline

Tiles de mapa são baixados preventivamente em formato MBTiles (SQLite). Cada tile é indexado por zoom/x/y e servido localmente via MapLibre sem necessidade de rede.

SQLite local Tiles z0–z17 GZIP comprimido
Trilhas em Tempo Real

Pontos GPS são gravados diretamente no Hive DB local a cada segundo. O buffer funciona sem rede. Ao gravar, o arquivo GPX é gerado localmente e enfileirado para sync.

1Hz logging Kalman filter GPX local
Waypoints & Coleta

Pontos criados, fotos capturadas e formulários preenchidos são salvos no Hive com UUID local. Nenhum dado é perdido. Ao conectar, o servidor resolve conflitos via timestamp.

UUID local Fotos em cache Zero perda
GeoPDF Offline

PDFs importados são processados pelo GDAL no dispositivo. As tiles são geradas localmente e armazenadas no MBTiles Store. O arquivo original fica no File System para re-geração.

GDAL on-device Tile pyramid Layer overlay
GPS Contínuo

Localização funciona 100% via hardware GPS do dispositivo — independente de rede. Modo background ativo mesmo com tela bloqueada. Bateria otimizada com adaptive sampling.

Hardware GPS Adaptive Hz Background mode
Segurança & Integridade

Todos os dados locais são encriptados com AES-256 via Flutter Secure Storage. Logs de operações garantem integridade. Fallback de dados críticos em múltiplas cópias locais.

AES-256 Op log Redundância

E se o PDF não tiver georreferência?
O app detecta a ausência de metadados geoespaciais e exibe um aviso claro. O arquivo ainda pode ser visualizado como imagem estática, mas não aparece como camada no mapa e não tem sobreposição GPS. O usuário pode inserir pontos de controle manualmente (Ground Control Points) para georreferenciar o arquivo.
