  allowHeaders: ['Content-Type','Authorization','X-Device-ID'],
  exposeHeaders: ['X-SW-Cache'],
  credentials: true,

CORS no Nginx (VPS) 
# Adicionar ao bloco /api/ no nginx.conf
location /api/ {
    proxy_pass http://localhost:3000/api/;

    # CORS headers para o Nginx também (caso API retorne erro antes do app)
    add_header 'Access-Control-Allow-Origin'  '$http_origin' always;
    add_header 'Access-Control-Allow-Methods' 'GET,POST,PUT,PATCH,DELETE,OPTIONS' always;
    add_header 'Access-Control-Allow-Headers' 'Content-Type,Authorization,X-Device-ID' always;
    add_header 'Access-Control-Allow-Credentials' 'true' always;

    if ($request_method = 'OPTIONS') {
        add_header 'Access-Control-Max-Age' 86400;
        return 204;
    }
}

# Tiles MBTiles — CORS aberto (tiles são públicos por ID)
location /tiles/ {
    proxy_pass http://localhost:3000/tiles/;
    add_header 'Access-Control-Allow-Origin' '*' always;
    add_header 'Cache-Control' 'public, max-age=2592000' always;
    proxy_cache_valid 200 30d;
}
Regra para tiles: O endpoint /tiles/:id/:z/:x/:y.png usa Access-Control-Allow-Origin: * porque o Leaflet carrega tiles como recursos de imagem (sem credenciais). Os endpoints /api/v1/* usam origem restrita + credentials.

Passo 4 — Configurar Nginx (frontend + proxy API)
# Criar config Nginx
cat > /etc/nginx/sites-available/afaiamaps <<'EOF'
server {
    listen 80;
    server_name seudominio.com www.seudominio.com;

    # Frontend PWA
    root /var/www/afaia-maps/frontend;
    index index.html;

    # Service Worker sem cache
    location = /sw.js {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        add_header Content-Type "application/javascript";
    }

    # Manifest sem cache
    location = /manifest.json {
        add_header Cache-Control "no-cache";
    }

    # Assets estáticos com cache longo
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # SPA fallback
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy API
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 5500M;
        proxy_read_timeout 300s;
    }

    # Proxy Tiles MBTiles
    location /tiles/ {
        proxy_pass http://localhost:3000/tiles/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        add_header 'Access-Control-Allow-Origin' '*' always;
        add_header 'Cache-Control' 'public, max-age=2592000' always;
        proxy_cache_valid 200 30d;
        proxy_read_timeout 30s;
    }

    # Uploads (arquivos MBTiles, PDFs)
    location /uploads/ {
        proxy_pass http://localhost:3000/uploads/;
        proxy_read_timeout 60s;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
}
EOF

ln -s /etc/nginx/sites-available/afaiamaps /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

  1. Acesse vercel.com/new → importe o repositório
  2. Configure: 
    Root Directory:      frontend
Framework Preset:    Other
Build Command:       (vazio — sem build step)
Output Directory:    .   (ponto = raiz do frontend/)
  3. Adicione o arquivo frontend/vercel.json: 
    {
  "headers": [
    {
      "source": "/sw.js",
      "headers": [
        { "key": "Cache-Control", "value": "no-cache, no-store, must-revalidate" },
        { "key": "Content-Type",  "value": "application/javascript" },
        { "key": "Service-Worker-Allowed", "value": "/" }
      ]
    },
    {
      "source": "/manifest.json",
      "headers": [{ "key": "Cache-Control", "value": "no-cache" }]
    },
    {
      "source": "/(.*\\.(?:js|css|png|jpg|ico|svg|woff2))",
      "headers": [{ "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }]
    }
  ],
  "rewrites": [
    { "source": "/((?!api|tiles|uploads).*)", "destination": "/index.html" }
  ]
}
  4. Deploy → obtenha a URL (ex: afaiamaps.vercel.app)
  5. Volte ao Railway e atualize FRONTEND_URL=https://afaiamaps.vercel.app
  6. Atualize o CORS no backend/src/index.js e faça redeploy
Ação necessária: Crie o arquivo frontend/vercel.json com o conteúdo acima antes do deploy no Vercel. Ele garante que o Service Worker seja servido com o Content-Type correto.

# 1. Edite os arquivos em frontend/
# 2. Sincronize com nativo
npx cap sync

# 3. Android: basta rebuildar no Android Studio
# 4. iOS: basta rebuildar no Xcode

# Atualização sem rebuild nativo (OTA via Service Worker):
# O SW v3 detecta nova versão automaticamente e atualiza o cache
# Útil para pequenas correções sem passar pelas lojas

# Verificar versão do SW ativo:
# DevTools → Application → Service Workers → "Source"
Permissões declaradas nos apps

Android (AndroidManifest.xml)

  • ACCESS_FINE_LOCATION — GPS
  • ACCESS_COARSE_LOCATION — localiz.
  • ACCESS_BACKGROUND_LOCATION — trilhas
  • CAMERA — fotos georreferenciadas
  • READ/WRITE_EXTERNAL_STORAGE — mapas
  • INTERNET — sync
  • FOREGROUND_SERVICE — GPS bg

iOS (Info.plist)

  • NSLocationWhenInUseUsage
  • NSLocationAlwaysUsage — trilhas
  • NSCameraUsageDescription
  • NSPhotoLibraryUsage
  • location background mode
